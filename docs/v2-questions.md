# Virtual Silicon Lab v2 — User Decisions

v2 계획을 쓰기 전에 제품 방향을 먼저 정해야 한다. 아래 질문에 직접 답을 적으면, 그 답을 기준으로 v2 spec/roadmap을 작성한다.

## 1. v2의 핵심 목표

**질문:** v2에서 가장 크게 좋아져야 하는 것은 무엇인가?

답: 비주얼 고도화가 v2에서 가장 큰 목표이다. images/ 폴더를 별도로 만들어서 칩 관련 이미지를 몇 개 넣어 놓았는데 이걸 참고하면 좋겠다. 세부 사항에 대해: 1번 목표는 웹 페이지의 전반적인 디자인 향상, 2번 목표는 칩 다이 편집 화면의 디자인 향상(실제 칩 느낌이 나도록), 3번 목표는 출력되는 이미지의 디자인 변경이다. 이외에 선택지에 제시한 사항들은 모두 추후 작업할 예정이다.

선택지 참고:
- 공유/커뮤니티: 만든 칩을 링크로 공유하고 다른 사람이 볼 수 있게 한다.
- 비주얼 고도화: 더 멋진 shader/glow/texture/가짜 3D를 만든다.
- 창작 자유도: custom die, 더 많은 장식, 레이아웃 편집을 강화한다.
- 사용성/완성도: 모바일/반응형, onboarding, export 품질, 안정성을 다듬는다.
- 세계관/콘텐츠: 칩 시리즈, lore, 가짜 벤치마크, spec generator를 확장한다.

## 2. v2의 성공 기준

**질문:** v2가 끝났다고 판단할 수 있는 구체적 기준은 무엇인가? 3~5개만 적어주세요.

답: 생성된 hero chip과 그 poster들이 images/ 폴더에 있는 실제 기업들이 보도자료로 내놓는 이미지와 유사한 품질을 보여야 한다. 또한 hero chip과 poster 10개 세트가 같은 품질 기준을 통과해야 한다.

예:
- 사용자가 URL 하나로 칩 poster를 공유할 수 있다.
- 새 사용자가 preset에서 poster를 만들고 공유하기까지 3분 이내다.
- hero chip 10개가 같은 품질 기준을 통과한다.

## 3. 공유와 백엔드

**질문:** v2에서 백엔드를 도입해도 되나요?

답: v2에서 백엔드를 도입한다. v2에서 DB는 SQLite를 사용하고, 추후 다른 DB로 변경한다. 로그인/회원가입/회원정보 수정/회원탈퇴 등 계정 관련 기능을 구현하되, 계정 더미 데이터는 관리자 계정 1개만 생성하는 것으로 한다. 공유의 경우 아직 이미지 공유만 허용하므로 링크는 의미가 없다.

정해야 할 것:
- 백엔드 없음 유지 / Supabase 등 도입 / GitHub Pages 같은 정적 배포 우선
- 계정 없이 익명 공유만 허용할지, 로그인/계정까지 갈지
- 편집 가능한 remix 링크가 필요한지, 읽기 전용 showcase 링크면 충분한지

## 4. 커뮤니티 범위

**질문:** gallery, ranking, contest 같은 커뮤니티 기능을 v2에 넣을까요?

답: gallery, ranking, contest 등의 기능을 포함하는 게시판 기능을 v2에서 구현한다. 더미 데이터는 테스트 용도로만 소량 만들어서 DB에 추가한다.

정해야 할 것:
- 개인 공유 링크까지만
- 공개 갤러리까지
- 좋아요/랭킹/콘테스트까지
- v2에서는 제외

## 5. 비주얼 기술 방향

**질문:** 고급 비주얼을 위해 PixiJS/Three.js/shader 계열을 도입해도 되나요?

답: 현재 구현되어 있는 2D texture와 preset의 품질을 올리되, 추후 Three.js 등 진짜 3D를 도입할 수 있는 확장 가능한 구조로 구현한다.

정해야 할 것:
- Konva만 유지
- CSS 기반 가짜 3D만 추가
- PixiJS 필터 도입
- Three.js 진짜 3D 도입
- shader는 하지 않고 texture/preset 품질만 올림

## 6. 모바일/반응형

**질문:** v2에서 모바일을 지원해야 하나요?

답: v2에서는 모바일 환경을 지원하지 않는다. 데스크탑 웹 페이지 구현에 집중한다.

정해야 할 것:
- 데스크탑 전용 유지
- 모바일은 viewer/share만 지원
- 모바일에서도 editor 가능해야 함
- 태블릿 우선 지원

## 7. 창작 자유도

**질문:** 사용자가 직접 바꿀 수 있는 범위를 얼마나 넓힐까요?

답: v1처럼 preset remix 중심 구조를 유지하되, 향후 자유 편집 기능을 추가할 수 있다는 것을 인지한다.

정해야 할 것:
- custom die size/path
- block color/label/type custom
- decoration 편집/삭제/drag
- multi-select/alignment
- freeform drawing/routing
- v1처럼 preset remix 중심 유지

## 8. AI 기능

**질문:** AI 생성/추천 기능을 v2에 넣을까요?

답: v2에는 AI 기능을 넣지 않고, AI가 필요하지 않은 정도의 무작위 chip generator만 구현한다.

정해야 할 것:
- AI 없음
- 랜덤 chip generator만
- prompt 기반 chip/spec 생성
- style/theme 추천
- poster copy/lore 생성

## 9. 세계관과 콘텐츠 톤

**질문:** v2의 톤은 v1처럼 병맛 Sci-Fi를 유지할까요, 아니면 더 진지한 포트폴리오 톤으로 갈까요?

답: v1의 웹 페이지 분위기는 딱딱한 Sci-Fi이다. 구석에 별도의 페이지 테마 전환 버튼을 만들고, 연구실/아니메/우주 테마를 추가해서 사용자가 직접 변경할 수 있도록 한다. 기본 테마는 연구실이다.

정해야 할 것:
- v1보다 더 웃기고 초현실적으로
- Apple keynote/portfolio 톤 강화
- retro computing/zine 톤 강화
- military/industrial 톤 강화
- 사용자가 직접 tone preset 선택

## 10. v1에서 반드시 유지할 것

**질문:** v2에서도 절대 망가뜨리면 안 되는 v1 경험은 무엇인가?

답: 없다. v2에서 더 개선될 수 있다면 v1의 모든 것을 수정해도 상관없다.

예:
- 로그인 없이 바로 시작
- 5분 안에 멋진 결과
- 로컬 저장
- poster PNG 품질
- EDA 툴처럼 보이지 않는 것

## 11. v1에서 과감히 바꿔도 되는 것

**질문:** v2에서 구조나 UX를 바꾸기 위해 깨도 되는 v1 결정은 무엇인가?

답: 단일 JSON schema는 일단 유지한다. 단일 JSON schema를 버리고 다른 형식으로 변경해야 하는 경우 수정하기 전에 먼저 의사를 묻는다.

예:
- local-only 저장
- desktop-only
- Konva-only rendering
- 단일 JSON schema
- 현재 landing/dashboard/editor layout

## 12. v2의 크기와 일정

**질문:** v2는 어느 정도 크기로 진행할까요?

답: 비주얼 부분을 완전히 다른 수준으로 개선해야 하므로 새로운 major version으로 만드는 것이 좋다.

선택지 참고:
- 작게: 1~2주짜리 polish release
- 중간: 3~5개 milestone
- 크게: 공유/커뮤니티/비주얼까지 포함한 새 major version

## 13. 우선순위에서 제외할 것

**질문:** v2에서 명시적으로 하지 않을 것은 무엇인가?

답: 모바일과 관련된 모든 기능, 진짜 3D 렌더링, AI 기능, 결제/수익화 등은 명시적으로 하지 않는다. 실제 반도체 호환성은 모든 버전에서 할 생각이 없다.

예:
- 로그인
- 모바일 editor
- 진짜 3D
- AI
- 공개 갤러리
- 결제/수익화
- 실제 반도체 호환성

## 14. 배포 목표

**질문:** v2 완료 후 어디에 공개할 계획인가요?

답: 비공개

정해야 할 것:
- 로컬 포트폴리오만
- Netlify/Vercel 공개 URL
- GitHub Pages
- README/demo 중심
- 실제 사용자를 받을 공개 서비스

## 15. 개인적으로 가장 만들고 싶은 기능

**질문:** 기술/제품 타당성과 별개로, v2에서 제일 만들고 싶은 기능 하나는 무엇인가요?

답: images/ 폴더에 있는 이미지들처럼 chip editor 페이지가 멋있게 보이는 것을 만들고 싶다. 칩 다이를 세세하게 묘사할 필요는 없다.
